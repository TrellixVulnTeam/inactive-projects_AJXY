URL Methode Parameter Rueckgabewert Beschreibung
/account POST Die Id des neuen Accounts Legt einen neuen Account an.
/account PUT Aktualisiert einen Account.
/account GET Eine Liste JSON Daten über alle Accounts. Liest alle Accounts aus.
/account/{id} GET Die kompletten JSON Daten des Accounts Beispiel: {"id":0,"birthyear":1991,"sex":"m", "username":"John","city":"berlin","password":"passwort", "email":"testmail@gmail.com"} Liest einen Account anhand einer ID aus.
/account/{id} DELETE Einzelnen Nutzer anhand der ID löschen
/account/auth GET Wenn Password und Nutzername in Kombination in der Datenbank gefunden werden gibts true zurück. Ansonsten kommt false zurück. Über diese URL können kann festgestellt werden, ob Passwort und Nutzername zu einem legitimen Nutzer gehören.
/category POST Die Id der neuen Category. Legt eine neue Category auf der Datenbank an.
/category/{id} GET Die Category im Jsonformat. Beispiel: {"id":191,"text":"FASHION"} Liest eine Category anhand ihrer Id aus.
/category/{id} DELETE Löscht eine Kategorie anhand einer entsprechenden Id.
/hashtag POST Die Id des neu erstellten hashtags. Erstellt einen neuen Hashtag in der Datenbank.
/hashtag/{id} GET Die Daten des Hashtags im Json Format {"id":243,"text":"#hashtag"} Liest den Text eines Hashtags anhand seiner ID aus
/hashtag GET Eine Liste alle Hashtags im Json Format Liest alle Hashtags aus.
/hashtag/{id} DELETE --- Löscht einen Hashtag anhand einer ID.
/project POST Id des gespeicherten Projects Speichert ein neues Projekt auf der Datenbank.
/project PUT Aktualisiert ein bestehendes Projekt. Wie bei der Projekterstelltung muss das komplette Json an die Rest API übergeben werden.
/project GET Eine JSON Liste aller bislang angelegten Projekte Liest alle bestehenden Projekte aus.
/project/{id} GET Die Json Daten des Projekts {"id":1,"name":"project1", "location":"berlin1", "description":"beschreibung1", "specialFeatures":"Das macht es so besonders!1", "motivation":"motivation1", "expirationDate":1456950854730, "accountId":1,"hashtags":[]} Liest ein bestimmtes Projekt anhand einer ProjektId aus.
/project/{id} DELETE Löscht ein bestimmtes Projekt anhand seiner ProjektId
/project/{projectId}/hashtags/{hashtagId} POST Fügt ein existierendes Hashtag anhand seiner Id einem bestimmten Projekt hinzu.
/project/{projectId/hashtags/{hashtagId}} DELETE Entfernt die Zuordnung eines Hashtag zu einem bestimmten Projekt anhand der ProjectId und der Hashtag Id.
/project/{projectId}/categories/{categoryId} POST Ordnet eine existierende Kategory anhand der Kategory ID einem Projekt zu.
/project/{projectId}/categories/{categoryId} DELETE Löscht die Zuordnung einer Kategory zu einem Projekt anhand der Id's
/project/{projectId}/uploads/{uploadId} POST Ordnet einem Project einen Upload anhand der UploadId zu.
/project/{projectId}/uploads/{uploadId} DELETE Löscht die Zuordnung eines Uploads zu einem Projekt anhand der Upload und der Project Id
/upload/{userId} POST Eine Liste der Dateinamen und ihrer zugeordneten Id's Läd eine Menge von Dateien auf den Server. Die Datei selber wird im Verzeichnis Upload gespeichert und es wird ein Datensatz in der Uploadtabelle angelegt. Die Datei bekommt als neuen Namen die ID aus der Datenbank.
/upload/{uploadId} DELETE Löscht eine Datei und ihren zugehörigen Datensatz.
/upload/path/{uploadId} GET Der realtive Pfad auf dem Server Liest den Dateipfad auf dem Server zur Datei aus.
